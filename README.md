# Novel Shelf

Novel Shelfは、小説家になろう・カクヨム・ハーメルンなど複数のWeb小説をまとめて管理できる、ブラウザだけで動く軽量な本棚管理ツールです。

「どの作品をどこまで読んだか分からない」「更新された作品だけ追いたい」「投稿サイトごとにブックマークが散らばる」といった悩みを、ひとつの本棚で整理できます。本文データは取得・保存せず、作品URL、更新話数、読了位置、メモなど管理に必要な情報だけをブラウザ内に保存します。

公開ページ: https://kirica-9029.github.io/novel-reader/

## 主な機能

- 複数サイトのWeb小説を本棚に登録
- 小説家になろうAPI検索から作品を追加
- カクヨム、ハーメルン、Arcadia、暁、pixiv小説、ノベルアップ+などを外部リンク管理
- 小説家になろう作品の更新確認
- 読了位置、未読話数、差分の管理
- 本棚内検索、サイト別・状態別フィルター、並び替え
- お気に入り登録
- 更新リストのドラッグ&ドロップ並び替え
- JSONエクスポート/インポートによるバックアップ
- PWA対応による最低限のオフライン表示
- スマホ対応のレスポンシブUI

## 利用方法

1. 公開ページを開きます。
2. 「作品URLを登録」または「小説家になろうで検索」から作品を追加します。
3. 本棚で読了位置、メモ、お気に入りを管理します。
4. 更新タブで小説家になろう作品の更新確認を実行します。
5. 設定タブから定期的にJSONを書き出してバックアップします。

データは利用中のブラウザのlocalStorageに保存されます。端末変更、別ブラウザ利用、ブラウザデータ削除時には自動で引き継がれないため、JSONバックアップを推奨します。

## 動作確認手順

### 検索画面

- 検索画面を開く
- 詳細検索を展開/折りたたみする
- ジャンルを指定して検索する
- 並び順を変更して検索する
- 検索結果から本棚追加する

### ランキング画面

- ランキング画面を開く
- 日間/週間/月間/四半期/年間/累計を切り替える
- ジャンルを切り替える
- 各ランキング結果が最大100件程度まで表示される
- 順位が1位から連番で表示され、ジャンル切り替え後も飛び番にならないことを確認する
- API失敗時にローディングが残らないことを確認する

## 開発環境のセットアップ

このリポジトリはビルドなしで動く静的サイトです。

```bash
git clone https://github.com/kirica-9029/novel-reader.git
cd novel-reader
```

ローカルサーバーを起動します。

```bash
py -m http.server 8080 --bind 127.0.0.1
```

ブラウザで開きます。

```text
http://127.0.0.1:8080/
```

Service WorkerやPWA挙動は、`file://` ではなく `localhost` または `127.0.0.1` で確認してください。

## 注意事項

- Web小説本文は取得・保存しません。
- データはブラウザ内のlocalStorageに保存されます。
- 小説投稿サイトの仕様変更により、一部機能が動作しなくなる場合があります。
- 各投稿サイトの利用規約に従って利用してください。
