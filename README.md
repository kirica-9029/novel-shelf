# Novel Shelf

Novel Shelfは、小説家になろう・カクヨム・ハーメルンなど複数のWeb小説をまとめて管理できる、ブラウザだけで動く軽量な本棚管理ツールです。

「どの作品をどこまで読んだか分からない」「更新された作品だけ追いたい」「投稿サイトごとにブックマークが散らばる」といった悩みを、ひとつの本棚で整理できます。本文データは取得・保存せず、作品URL、更新話数、読了位置、メモなど管理に必要な情報だけをブラウザ内に保存します。

公開ページ: https://kirica-9029.github.io/novel-reader/

## 現在の主要機能

- 複数サイトのWeb小説を本棚に登録
- 小説家になろう公式API検索から作品を追加
- 詳細検索によるジャンル、並び順、連載/完結、取得件数の指定
- カクヨム、ハーメルン、Arcadia、暁、pixiv小説、ノベルアップ+、ノクターンの外部検索・URL登録
- 小説家になろう作品の更新確認
- 読了位置、未読話数、更新差分、メモ、お気に入りの管理
- 本棚内検索、サイト別・状態別フィルター、並び替え
- 更新リストのドラッグ&ドロップ並び替え
- 小説家になろうランキングの日間、週間、月間、四半期取得
- ランキングのジャンル切り替え
- ランキング結果から本棚へ追加
- JSONエクスポート/インポートによるバックアップ
- 読了登録ブックマークレット
- PWA対応による最低限のオフライン表示
- スマホ対応のレスポンシブUI

## 利用方法

1. 公開ページを開きます。
2. 「作品URLを登録」または「小説家になろうで検索」から作品を追加します。
3. 必要に応じて詳細検索を開き、ジャンルや並び順を指定します。
4. 本棚で読了位置、メモ、お気に入りを管理します。
5. 更新タブで小説家になろう作品の更新確認を実行します。
6. ランキングタブで小説家になろうのランキングを確認し、気になる作品を本棚へ追加します。
7. 設定タブから定期的にJSONを書き出してバックアップします。

データは利用中のブラウザのlocalStorageに保存されます。端末変更、別ブラウザ利用、ブラウザデータ削除時には自動で引き継がれないため、JSONバックアップを推奨します。

## 既知の制約

- Web小説本文は取得・保存しません。
- データはブラウザ内のlocalStorageに保存されます。
- 端末変更、別ブラウザ利用、ブラウザデータ削除時には自動同期されません。
- API検索、ランキング取得、更新確認は小説家になろうのみ対応しています。
- 小説家になろう以外のサイトは外部検索とURL登録による手動管理です。
- 外部サイトの仕様変更、URL形式変更、API制限により一部機能が動作しなくなる場合があります。
- Service Workerの影響で、更新直後に古いファイルが残ることがあります。表示がおかしい場合は再読み込みやキャッシュ削除を試してください。
- PWAのオフライン表示は最低限のアプリ表示を目的としており、外部API取得はオンライン時のみ動作します。
- 広告枠は将来的なGoogle AdSense等の導入を想定しています。広告配信時はプライバシーポリシーに従って運用します。

## 広告・アフィリエイト導線

- アプリ画面と各情報ページに広告掲載予定スペースを配置しています。
- フッターには、プライバシーポリシー、利用規約、要望・お問い合わせ、運営者情報への導線があります。
- 広告やアフィリエイトを導入する場合は、掲載内容が各小説投稿サイトの利用規約や広告ポリシーに反しないことを確認します。
- 広告配信、アクセス解析、Cookie等の扱いはプライバシーポリシーに記載します。

## 開発環境のセットアップ

このリポジトリはビルドなしで動く静的サイトです。

```bash
git clone https://github.com/kirica-9029/novel-reader.git
cd novel-reader
```

ローカルサーバーを起動します。

```bash
py -m http.server 8080 --bind 127.0.0.1
```

ブラウザで開きます。

```text
http://127.0.0.1:8080/
```

Service WorkerやPWA挙動は、`file://` ではなく `localhost` または `127.0.0.1` で確認してください。
